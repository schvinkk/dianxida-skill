# 滇西应用技术大学毕业论文排版 Skill

## 核心原则：复制粘贴，不要自创

**绝对不要从零创建格式。** 必须以例文文档为基底，复制其 XML 结构，只替换内容。例文已经包含了学校要求的所有样式定义（字体、段落、间距、页眉页脚样式），从零创建必然出错。

## 铁律：样式 ID 因文档而异，必须动态分析

**绝不能硬编码样式 ID。** 不同文档的样式 ID 完全不同，即使是同一模板派生的文档也会因编辑历史产生差异。

四文档样式 ID 对比（实测数据）：

| 样式名称 | 模板(.docx) | 例文 | 1.毕业论文+学校正确格式 |
|----------|:-----------:|:----:|:----------------:|
| Normal（正文） | 1 | 1 | 1 |
| heading 1（一级标题） | 2 | 2 | 2 |
| heading 2（二级标题） | 3 | 3 | 3 |
| heading 3（三级标题） | 4 | 4 | 4 |
| toc 1（一级目录） | **8** | **10** | **9** |
| toc 2（二级目录） | **7** | **11** | **8** |
| toc 3（三级目录） | **6** | **7** | **7** |
| header（页眉样式） | **11** | **9** | **12** |
| footer（页脚样式） | **10** | **8** | **11** |
| Plain Text | 9 | 12 | **10** |
| Title | 12 | — | **14** |
| Body Text Indent | 5 | 6 | **6** |
| TOC 标题1（自定义） | 15 | — | **19** |

**操作前必须运行分析脚本**，从例文的 `styles.xml` 中按样式名称查找实际 ID，再用这些 ID 进行后续操作。详见 [ooxml-reference.md](ooxml-reference.md) 中的动态分析脚本。

### 关键发现：样式定义 vs 实际 run 覆盖

**样式定义中的字号 ≠ 段落中实际显示的字号。** 论文中 run 级别的 `<w:sz>` 会覆盖样式定义。排版时必须按 run 级别的实际值来设置：

| 样式 | 样式定义中的 sz | 实际 run 覆盖 sz | 实际字号 |
|------|:--------------:|:----------------:|:-------:|
| heading 1 | sz=30 (15pt) | **sz=32 (16pt = 三号)** | 三号 |
| heading 2 | sz=36 (18pt) | **sz=28 (14pt = 四号)** | 四号 |
| heading 3 | sz=28 (14pt) | **sz=24 (12pt = 小四)** | 小四 |
| toc 1 | sz=30 (15pt) | **sz=24 (12pt = 小四)** | 小四 |
| toc 2 | sz=28 (14pt) | **sz=24 (12pt = 小四)** | 小四 |
| toc 3 | — | **sz=24 (12pt = 小四)** | 小四 |
| header | sz=18 (9pt) | **sz=21 (10.5pt = 五号)** | 五号 |
| footer | sz=18 (9pt) | **sz=21 (10.5pt = 五号)** | 五号 |

**结论：所有格式操作应以 run 级别的实际值为准，不能只看样式定义。**

## 铁律：绝不修改用户的文字内容

**排版只改格式，不动内容。** 用户写了什么就是什么，绝不能擅自修改、删减、润色或重写用户的论文正文。

具体规则：

- 正文段落、标题文字、摘要内容、关键词 — **原样保留**，只调整字体、字号、缩进等格式属性
- 唯一允许的文字变更：标题中的空格规范化（"摘要"→"摘  要"、"目录"→"目  录"、"致谢"→"致  谢"）
- 封面信息（学院、专业、姓名等）— 从草稿中提取后填入，不改写
- 参考文献条目 — 原样保留，只调格式（字号、字体、缩进、标点规范化）
- 如果用户的文字有明显的错别字或语法问题 — **不直接修改**，而是在输出后以清单形式提醒用户自行修正

**违反此规则等同于破坏用户论文，是最高级别的错误。**

## 附加任务：AI 率检查与优化

排版完成后，必须帮用户评估论文的 AI 生成率（AI detection rate），并给出优化建议：

### 检查方法

1. **提取全文正文**（不含封面、声明、目录、参考文献），逐段分析
2. **识别 AI 痕迹特征**：
   - 过度使用"首先...其次...最后..."等模板化连接词
   - 段落开头千篇一律的总结句
   - 过于工整的并列结构（三个以上的并列短句）
   - 空洞的过渡句（"综上所述"、"总而言之"、"由此可见"高频出现）
   - 学术腔过重但缺乏具体数据/实验细节支撑
   - 同义词替换过于精准（人写作时不会每段都换不同说法）
3. **逐段评分**：每段给出 AI 嫌疑等级（低/中/高）
4. **输出诊断报告**：列出所有"高风险"段落及具体问题

### 优化建议

对 AI 率偏高的段落，提供具体的改写方向（**不直接改写，只给建议**）：

- 加入个人实验数据、具体数值、本校特色的表述
- 打破过于工整的并列结构，用口语化但不失学术的表达
- 减少模板化连接词，换成更自然的过渡
- 插入引用具体文献的讨论（"正如张三(2023)的研究所示..."）
- 适当加入限定性表述（"在本实验条件下"、"基于滇西地区的实际情况"）

### 输出格式

```
=== AI 率诊断报告 ===
总体评估：中/高/低
高风险段落：
  - 第X章第X节（para N）：[问题描述] → [改写建议]
  - ...
建议用户重点修改的段落数量：X 段
```

## 附加任务：内容完善提醒

排版过程中，如果发现论文内容有以下问题，在输出后以清单形式提醒用户：

- **缺失部分**：缺少摘要/Abstract/关键词/致谢/参考文献等必要章节
- **格式提醒**：图表缺少标题或编号、公式缺少编号、参考文献引用格式不统一
- **内容薄弱**：某个章节明显篇幅过短（如"结论"只有一两句话）、讨论部分缺乏深度分析
- **数据缺失**：实验结果部分缺少具体数据、表格或图表
- **引用问题**：正文中出现引用标记但参考文献列表中没有对应条目，或反之
- **中英文摘要不匹配**：中文摘要和 Abstract 内容明显不对应

**注意：只提醒，不代改。内容完善是用户自己的事。**

## 学校格式要求汇总（模板17条规则提炼）

以下规则从模板文档的"格式要求1-17"中提取，是排版的权威依据：

### 封面
- 格式要求1：中文标题 — 宋体，三号(16pt)，加粗，居中
- 格式要求2：英文标题 — Times New Roman，三号，加粗，居中
- 格式要求3：学生信息栏 — 宋体，四号(14pt)，日期格式如"2023年10月1日-2024年6月15日"

### 摘要与关键词
- 格式要求4："摘  要"标题 — 字间空2格，黑体，三号，加粗，居中，段前12磅段后6磅
- 摘要正文 — 首行缩进2字符，宋体，小四(12pt)
- "关键词"标签 — 首行缩进2字符，黑体，小四，加粗

### Abstract
- 格式要求6："Abstract"标题 — Times New Roman，三号，加粗，居中，段前12磅段后6磅
- Abstract正文 — 首行缩进2字符，Times New Roman，小四
- "Key Words"标签 — 首行缩进2字符，Times New Roman，小四，加粗

### 目录
- 格式要求7："目  录"标题 — 字间空2格，黑体，三号，加粗，居中
- 一级目录 — 顶格，黑体，小四，右对齐
- 二级目录 — 首行缩进2字符，宋体，小四，右对齐
- 三级目录 — 首行缩进4字符，宋体，小四，右对齐

### 正文
- 格式要求8：一级标题(heading 1) — 黑体，三号，加粗，居中，段前12磅段后6磅
- 二级标题(heading 2) — 黑体，四号，加粗，左对齐，段前6磅段后0磅
- 三级标题(heading 3) — 黑体，小四，加粗，左对齐
- 正文 — 宋体/Times New Roman，小四，两端对齐，首行缩进2字符

### 页眉页脚
- 格式要求5：页码 — 页脚方式，五号，页面底端居中；摘要/Abstract/目录用罗马数字各自编号，正文用阿拉伯数字
- 格式要求10：页眉 — 从正文第1页开始，内容为当前页完整一级标题文字，宋体，五号，居中

### 图表公式
- 格式要求11：图序和图题 — 空一格，置于图下方居中，中文黑体/数字TNR，小四
- 格式要求12：表序和表题 — 空一格，置于表格上方居中，中文黑体/数字TNR，小四；表格内容宋体/TNR五号
- 格式要求13：三线表 — 各栏标注量和单位，不用"同上"/"同左"
- 格式要求14：公式 — 另起一行居中，编号在右侧行末，TNR小四

### 参考文献
- 格式要求9：引用 — 按顺序编号，序号置于引用部分右上角，TNR小四
- 格式要求16：标题同heading 1；正文中文宋体/英文TNR，五号，逗号句号用半角TNR

### 致谢/附录
- 格式要求15：标题 — 字间空2格，字体字号同heading 1

### 页面设置
- 格式要求17：A4幅面，页边距上下2.5cm/左3.0cm/右2.0cm，行距多倍1.25，双面印刷

### Step 0: 动态分析（必须最先执行）

在操作任何 XML 之前，**必须**运行动态分析脚本，从例文和模板的 `styles.xml` 中提取以下信息：

```python
# 按样式名称查找实际 ID
style_name_map = {}
for style in styles_root.findall(f'{wns}style'):
    name_el = style.find(f'{wns}name')
    if name_el is not None:
        style_name_map[name_el.get(f'{wns}val')] = style.get(f'{wns}styleId')

# 关键样式查找
heading1_id = style_name_map.get('heading 1')   # 通常为 2
header_style_id = style_name_map.get('header')   # 可能为 9/11/12
footer_style_id = style_name_map.get('footer')   # 可能为 8/10/11
toc1_id = style_name_map.get('toc 1')            # 可能为 8/9/10
toc2_id = style_name_map.get('toc 2')            # 可能为 7/8/11
toc3_id = style_name_map.get('toc 3')            # 可能为 6/7
```

后续所有涉及样式 ID 的操作，都必须用这里查到的动态值，不能写死。

### Step 1: 解压分析

```python
import zipfile, os
for name, path in [('example', '例文.docx'), ('template', '模板.docx'), ('review', '草稿.docx')]:
    dest = f'{name}_unpacked'
    with zipfile.ZipFile(path) as z:
        z.extractall(dest)
```

用 `xml.etree.ElementTree` 解析 `word/document.xml`，遍历所有 `<w:p>` 段落，记录：每个段落的序号、文本内容、样式 ID（`pStyle`）、是否包含 `sectPr`（分节符）。

**关键：必须先跑一遍分析脚本，搞清楚例文里每个分节符的位置和对应的页眉内容。** 参考 [ooxml-reference.md](ooxml-reference.md) 中的完整分析脚本。

### Step 2: 保留例文前置部分

例文的前 N 个段落（封面页、独创性声明、版权授权书等）直接保留，不替换。这些段落的格式完全正确，不要动。

通常例文在某个段落开始正文内容（摘要），找到这个分界点，保留之前的所有段落。

**前置部分结构（实测）：**
- 封面页：校名(方正小标宋_GBK sz=44) → 空行 → 论文标题(宋体 sz=32 加粗居中) → 英文标题(Times New Roman sz=32 加粗居中) → 学院/专业/班级/姓名/学号/指导教师/日期(宋体 sz=28)
- 独创性声明：标题(黑体 sz=30 加粗居中) → 正文(宋体 sz=24 首行缩进2字符) → 签名行
- 版权授权书：标题(黑体 sz=30 加粗居中) → 正文(宋体 sz=24) → 签名行

### Step 3: 封面标题

例文的封面页通常有两个标题段落（中文标题、英文标题），位置固定。找到它们后：

- 中文标题：宋体三号加粗居中（`sz=32, szCs=32, b, jc=center`）
- 英文标题：Times New Roman 三号加粗居中（`rFonts ascii/hAnsi="Times New Roman"`）

只替换 `<w:t>` 中的文本，保留原有 `<w:rPr>` 格式属性。

### Step 4: 摘要 / Abstract / 目录

从论文草稿中提取这三个部分，注意：

- "摘要" 改为 "摘  要"（中间两个全角空格）
- "目录" 改为 "目  录"
- 摘要标题样式：黑体三号加粗居中，段前12磅段后6磅（实测发现论文用 Title 样式来定义）
- Abstract 标题：Times New Roman 三号加粗居中，heading 1 样式
- 目录标题：黑体三号加粗居中
- 目录条目用域代码格式（见 Step 6），替换掉草稿中的手动目录

### Step 5: 分节符与页眉（最关键）

> **这是整个排版中最容易出错的部分。**

详见 [ooxml-reference.md](ooxml-reference.md) 中的完整说明。核心规则：

#### 规则 1：sectPr 放在当前 section 的最后一段

分节符 `<w:sectPr>` 定义的是**到此段落为止的 section** 的属性。所以：

- 摘要 section → sectPr 放在摘要内容的**最后一段**上
- 引言 section → sectPr 放在引言最后一段上
- 以此类推

**错误做法**：在新插入的空白段落上放 sectPr，用下一节的 header — 这会导致页眉错位一节。

**正确做法**：当遇到下一个一级标题时，把 sectPr 应用到 review_content 中该标题**之前**的最后一段。

```python
# 正确逻辑
if next_para_index in sect_end_map:
    _, hdr_rid, ftr_rid, pgfmt, pgstart = sect_end_map[next_para_index]
    # 找到 review_content 中的上一段
    target = review_content[-1]  # 或向上跳过合成段落
    set_sect_break(target, hdr_rid, ftr_rid, pgfmt, pgstart)
```

#### 规则 2：每个 section 的 header 显示该 section 自己的标题

| Section | 页眉内容 |
|---------|---------|
| 封面到目录 | 无页眉（空白 header） |
| 引言 | "引言"（去掉编号） |
| 资料与方法 | "资料与方法" |
| ... | （去掉章节编号） |
| 参考文献 | "参考文献" |
| 附录（如有） | "附录" |
| 致谢 | "致谢" |

**注意**：页眉显示的是**当前 section** 的一级标题，不是下一个 section 的。页眉从正文第一页（引言）开始设置，前置部分（摘要/Abstract/目录）不设页眉。

#### 规则 3：最后一个 section 由 body 级 sectPr 定义

文档最后一个 section（致谢）的页眉页脚由 `<w:body>` 直接子元素 `<w:sectPr>` 定义，不在段落上。

#### 规则 4：命名空间

headerReference 和 footerReference 的 `type` 属性必须用 relationships 命名空间 `r:type`，不是 `w:type`：

```python
hr.set(f'{rns}type', 'default')  # rns = relationships namespace
# 绝不能写 hr.set(f'{wns}type', 'default')
```

#### 规则 5：页码格式（实测验证）

| Section | pgNumType | 说明 |
|---------|-----------|------|
| 封面 section | `upperRoman start=1` | 封面不显示页码 |
| 声明+授权 section | 不设（继承 decimal） | 不显示页码 |
| 摘要 section | `upperRoman start=1` | 摘要页 = I |
| Abstract section | `upperRoman start=1` | Abstract = I |
| 目录 section | `upperRoman`（不设start） | 继承上节 = II, III... |
| 引言 section | `decimal start=1` | 正文第一页 = 1 |
| 后续正文各节 | `decimal`（不设start） | 连续编号 |

**注意**：每个 section 的 sectPr 上设置的 pgNumType 控制的是**该 section 自身**的页码格式。

### Step 6: 域代码目录

目录条目必须是域代码格式（不是纯文本加点号），匹配例文结构：

```
TOC \o "1-3" \h \u
  ├─ HYPERLINK \l _Toc{id} → [标题文本] → TAB → PAGEREF _Toc{id} \h
  ├─ ...（每个条目一行）
  └─ TOC field end
```

- 一级标题用 toc 1 样式（**动态 ID，从 styles.xml 查**）
- 二级标题用 toc 2 样式
- 三级标题用 toc 3 样式
- 制表位：右对齐，点线前导，`pos` 值从例文的 toc 1 样式定义中提取（论文中为 `pos="8777"`，不同文档可能不同）
- 外层用 HYPERLINK 域，内层用 PAGEREF 域

**目录条目的字体规则（实测）：**
- 一级目录：黑体（数字/英文用 Times New Roman），小四(sz=24)，右对齐
- 二级目录：宋体（数字/英文用 Times New Roman），小四(sz=24)，首行缩进2字符，右对齐
- 三级目录：宋体（数字/英文用 Times New Roman），小四(sz=24)，首行缩进4字符，右对齐

### Step 7: 参考文献格式

参考文献格式要求（格式要求16）：

- 标题"参考文献"用 heading 1 样式（黑体三号加粗居中）
- 条目字号为五号（`sz=21, szCs=21`，即 10.5pt）
- 中文用宋体，西文用 Times New Roman（`rFonts ascii/hAnsi="Times New Roman"`）
- **逗号、句号用半角**（TNR），不用全角标点
- 去掉首行缩进（`ind firstLine="0", firstLineChars="0"`）
- 两端对齐（`jc=both`）
- 行距：`line=300 auto`（多倍行距1.25倍）
- 无悬挂缩进（`left=0`）


在 `[Content_Types].xml` 中添加 Override 条目。

## 踩坑总结

详见 [pitfalls-and-fixes.md](pitfalls-and-fixes.md)，常见错误：

1. **硬编码样式 ID** — 不同文档 ID 不同，必须动态分析
2. **页眉错位一节** — sectPr 放在了新空白段上，应该放在 section 最后一段
3. **封面标题为空** — 只清空了文本没填入新标题
4. **目录是纯文本** — 没有用域代码格式
5. **页眉不显示** — `r:type` 写成了 `w:type`
6. **多出空白页** — 分节符和分页符叠加
7. **自创格式** — 没有复制例文，自己写样式定义
8. **页眉/页脚 pStyle 写错** — 必须用例文中 header 样式的实际 ID
